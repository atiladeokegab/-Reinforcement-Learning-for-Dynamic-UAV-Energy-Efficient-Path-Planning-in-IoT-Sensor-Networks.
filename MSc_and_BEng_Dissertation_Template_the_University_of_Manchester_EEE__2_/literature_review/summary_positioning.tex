\subsection{Summary and Project Positioning}

The literature splits along a clean fault line: path planners ignore the protocol's failure
modes (creating the SF Monoculture), and protocol optimisers use static mathematics that
cannot adapt to mobility. Zorbas et al.~\cite{zorbas2025revisiting} name the gap directly,
concluding that ``reinforcement learning models could be employed to adapt over time to
evolving network conditions.''

\subsubsection{Simulation Tool Gap}

No existing open simulation framework models the complete causal chain required for
protocol-aware UAV control. Bor et al.'s LoRaSim~\cite{bor2016lora} provides the
foundational LoRaWAN collision model but assumes a static topology and omits both ADR
dynamics and the Capture Effect, making it unsuitable for mobile-gateway evaluation.
Xiong et al.'s FlyingLoRa~\cite{xiong2023flyinglora} introduces UAV trajectory planning
over LoRa networks but omits ADR entirely (treating each sensor's data rate as fixed) and
assumes mutual packet destruction on any intra-SF collision, the exact assumption that
Croce et al.~\cite{croce2020lora} empirically disprove. Zorbas et al.~\cite{zorbas2025revisiting}
model ADR convergence accurately but use a static MILP formulation that cannot be used as
a Gymnasium-compatible step environment for DRL training. No prior work therefore provides
a Gymnasium-compatible environment that simultaneously models EMA-ADR convergence, the
6\,dB Capture Effect threshold, Two-Ray path loss, log-normal shadowing, and asynchronous
duty cycles within a single step, the combination required for an agent to learn
position-induced SF-diversity exploitation.

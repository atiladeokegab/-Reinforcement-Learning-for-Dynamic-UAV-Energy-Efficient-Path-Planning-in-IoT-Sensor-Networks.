\subsection{Summary and Project Positioning}

The literature splits along a clean fault line: path planners ignore
the protocol's failure modes (creating poor SF Allocation), and
protocol optimisers use static mathematics that cannot adapt to
mobility. No existing open simulation framework models the complete
causal chain: LoRaSim~\cite{bor2016lora} provides the foundational
LoRaWAN collision model but assumes static topology and omits both
ADR dynamics and the Capture Effect; FlyingLoRa~\cite{xiong2023flyinglora}
introduces UAV trajectory planning but omits ADR entirely and
assumes mutual destruction on any intra-SF collision (the assumption
Croce et al.~\cite{croce2020lora} empirically disprove);
Zorbas et al.~\cite{zorbas2025revisiting} model ADR convergence
accurately but use a static MILP that cannot be a Gymnasium-compatible
step environment for DRL training. Table~\ref{tab:related_work}
positions this dissertation at the specific intersection no prior
work occupies.

The proposed framework treats UAV position as a direct control input
for the LoRaWAN protocol, the step that static
optimisation~\cite{zorbas2025revisiting} and energy-centric
heuristics~\cite{xiong2023flyinglora} both skip. Modelling the
Capture Effect distinguishes this simulation from all prior LoRaWAN
simulators that assume mutual destruction on intra-SF collisions.

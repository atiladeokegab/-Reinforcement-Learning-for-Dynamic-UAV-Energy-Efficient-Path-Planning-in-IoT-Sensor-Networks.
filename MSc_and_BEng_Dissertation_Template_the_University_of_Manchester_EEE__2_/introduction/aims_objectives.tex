\subsection{Aims and objectives}
\label{sec:aims}

This project builds a protocol-aware RL framework for UAV-assisted LoRaWAN data collection and uses it to diagnose, then fix, a specific architectural limit of flat-policy learners. Five deliverables:

\begin{enumerate}
    \item Build a Gymnasium simulator coupling Two-Ray path loss, log-normal shadowing, EMA-based ADR ($\lambda{=}0.1$), and the LoRa Capture Effect into a single per-step causal chain. No prior open simulator includes all four.
    \item Train a DQN agent under domain randomisation and a competence-gated curriculum across five $(\text{grid}, N)$ conditions from $100{\times}100$/$N{=}10$ to $500{\times}500$/$N{=}50$, benchmarked against Nearest-Sensor Greedy, SF-Aware Greedy, and a TSP Oracle at 50 seeds per condition.
    \item Run a spatial diagnostic to determine whether the DQN performance ceiling at $N{\ge}30$ is a training failure or an architectural property of the flat-MLP encoder.
    \item Design and evaluate a permutation-invariant Relational-RL policy (Deep-Sets encoder, masked sum-pooling, PPO) under matched environment and curriculum conditions, compared against the DQN at the same five scales.
    \item Isolate which protocol and observation features are load-bearing through a four-condition ablation at 50 seeds per condition.
\end{enumerate}

The results chapter measures how far permutation invariance closes the TSP Oracle gap.

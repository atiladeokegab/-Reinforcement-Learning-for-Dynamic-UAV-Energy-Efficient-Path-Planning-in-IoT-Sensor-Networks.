\label{app:repo}

The repository contains the custom Gymnasium environment, the DQN training scripts, the baseline agent implementations, and the evaluation and plotting scripts used to produce all figures and tables in this dissertation.

\subsection*{Installation Instructions}

The project uses \texttt{uv} for deterministic dependency management. After cloning the repository:

{\footnotesize
\begin{verbatim}
# 1. Install uv (cross-platform)
pip install uv

# 2. Create virtual environment and install all dependencies
uv sync

# 3. (Optional) GPU acceleration ,  CUDA 12.1
uv pip install torch==2.5.1+cu121 --index-url https://download.pytorch.org/whl/cu121

# 4. Verify installation
uv run python -c "import gymnasium, stable_baselines3; print('OK')"
\end{verbatim}
}

\noindent Key dependencies: \texttt{torch==2.5.1}, \texttt{stable-baselines3>=2.3}, \texttt{gymnasium>=0.29}, \texttt{numpy}, \texttt{matplotlib}, \texttt{seaborn}, \texttt{pandas}.

\subsection*{Technical Details: Path Loss Model}

The simulator implements the Two-Ray Ground Reflection model. The received power $P_r$ at horizontal distance $d$ is:
\begin{equation}
P_r = P_t G_t G_r \frac{h_t^2 h_r^2}{d^4},
\label{eq:two_ray}
\end{equation}
where $P_t$ is transmit power, $G_t, G_r$ are antenna gains, and $h_t = 1.5$\,m (sensor), $h_r = 100$\,m (UAV). For $d < d_\text{cross}$ (crossover distance) the free-space Friis model is used instead. Zero-mean Gaussian shadowing $\mathcal{N}(0,\,4\,\text{dB}^2)$ is added independently per link per timestep. The path loss exponent effectively equals~4 in the Two-Ray regime, consistent with standard outdoor LoRa deployments~\cite{alhourani2014optimal}.

\subsection*{Reproduction}

Full instructions for training the DQN agent and reproducing all evaluation results are in the repository \texttt{README.md}. All random seeds used in the evaluation ($\{42, 123, 256, 789, 1337, \ldots\}$, 20 total per condition) are hard-coded in the evaluation scripts to ensure bit-exact reproducibility.

All third-party code used in this project is clearly identified in the repository. The DQN implementation uses the Stable-Baselines3 library (MIT licence).

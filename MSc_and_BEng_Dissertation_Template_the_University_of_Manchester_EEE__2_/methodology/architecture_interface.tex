\subsection{Architecture and Interface}

The simulation is architected as a custom environment compliant with the Gymnasium (formerly
OpenAI Gym) interface~\cite{towers2024gymnasium}, facilitating seamless integration with
standard DRL libraries such as Stable-Baselines3~\cite{raffin2021stable}. The environment encapsulates the complex cyber-physical interactions into a
discrete time-stepping logic.

Each \lstinline{step(action)} call sequentially: (1) updates UAV position and decrements
battery; (2) recomputes RSSI via the Two-Ray model; (3) updates AoI counters, generates
Poisson packets, and advances EMA-ADR; (4) resolves collisions via Capture Effect; and
(5) computes the multi-objective reward $R_t$.

\subsubsection{Training Algorithm}

At each step the agent selects $a_t = \arg\max_a Q(s_t, a; \theta)$ (or a random
action with probability $\epsilon$), executes the Gymnasium step, stores the transition
$(s_t, a_t, r_t, s_{t+1})$ in the replay buffer $\mathcal{D}$, and updates $\theta$
by minimising the TD loss on a sampled minibatch. The target network $\hat{Q}$ is hard
updated every $C$ steps to stabilise training. The complete pseudocode is provided in
Appendix~\ref{app:hyperparams}.


\subsubsection{Hyperparameters}

The DQN was trained for $2\times10^6$ timesteps, learning rate $\alpha = 3\times10^{-4}$,
discount $\gamma = 0.99$, replay buffer of 150{,}000 transitions, and batch size~128.
A target network is updated every 1{,}000 steps. $\epsilon$-greedy exploration decays
from 1.0 to 0.05 over the first 25\% of training steps. The policy network is a 3-layer
MLP with hidden dimensions [512, 512, 256] and ReLU activations. Full hyperparameter
listings are provided in \autoref{app:hyperparams}.
